📝 基础用法

1. 日志级别

支持 6 种日志级别：TRACE, DEBUG, INFO, WARNING, ERROR, FATAL

logger.trace('Trace message');
logger.debug('Debug message');
logger.info('Info message');
logger.warning('Warning message');
logger.error('Error message');
logger.fatal('Fatal message');

2. 多参数输出 原生体验

像原生 console 一样传递多个参数，对象可在控制台交互展开

const user = { id: 1, name: 'John', role: 'admin' };
logger.info('User logged in:', user, { timestamp: Date.now() });

// 对象在控制台中可以点击展开查看详情

3. 时间戳格式

支持多种时间戳格式：iso, locale, time, unix 或自定义函数

// ISO 格式
new Logger({ timestamp: 'iso' });

// 自定义格式
new Logger({
  timestamp: (date) => date.toLocaleTimeString()
});

4. 颜色控制

在浏览器控制台中显示彩色日志，可动态开关

const logger = new Logger({ colors: true });
logger.enableColors();  // 启用
logger.disableColors(); // 禁用

5. 日志级别过滤

设置最低日志级别，过滤不需要的日志

// 只显示 WARNING 及以上级别
const logger = new Logger({ level: LogLevel.WARNING });

logger.debug('不会显示');
logger.warning('会显示');

🎯 实用示例

6. 命名空间

为不同模块创建独立的日志实例

const apiLogger = new Logger({ namespace: 'api' });
const dbLogger = new Logger({ namespace: 'database' });

apiLogger.info('Request received');
dbLogger.info('Query executed');

7. 子 Logger

从父 Logger 继承配置，创建层级化日志

const app = new Logger({ namespace: 'app' });
const http = app.child('http');
const auth = http.child('auth');

auth.info('User authenticated');
// 输出: [app:http:auth] [INFO] User authenticated

8. 中间件

使用中间件扩展日志功能

import { prefixMiddleware, redactMiddleware } from '@gvray/logger';

logger.use(prefixMiddleware('[API]'));
logger.use(redactMiddleware(['password', 'token']));

9. 链式调用

流畅的 API 设计，支持链式配置

const logger = new Logger()
  .setLevel(LogLevel.DEBUG)
  .setTimestamp('time')
  .enableColors()
  .setDepth(3);

logger.info('Configured!');
