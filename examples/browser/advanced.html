🏷️ Namespaces & Child Loggers

使用命名空间组织模块化应用的日志，支持无限层级嵌套

基础命名空间

为不同模块创建独立的日志实例

const app = new Logger({ 
  namespace: 'app' 
});
app.info('Application started');

子 Logger

从父 Logger 继承配置，创建层级化日志

const db = app.child('database');
const cache = app.child('cache');
db.info('Connected');

深度嵌套

支持任意深度的命名空间嵌套

const http = app.child('http');
const auth = http.child('auth');
const jwt = auth.child('jwt');

🔗 Middleware System

通过中间件扩展日志功能，支持链式处理

自定义中间件

编写自己的日志处理逻辑

logger.use((ctx, next) => {
  ctx.args = ['[Custom]', ...ctx.args];
  next();
});

中间件链

多个中间件按顺序执行

logger
  .use(prefixMiddleware('[API]'))
  .use(timestampMiddleware())
  .use(filterMiddleware());

条件中间件

根据条件决定是否执行日志

logger.use((ctx, next) => {
  if (ctx.level >= LogLevel.ERROR) {
    // 只处理错误日志
    next();
  }
});
